from django.views.decorators.csrf import csrf_exempt, csrf_protect

@csrf_exempt   #给单个视图函数加上，那这个函数的跨站传输就不用校验，排除校验
@csrf_protect  #给单个视图函数加上，无论CSRF中间件是否有开启，都会强制这个函数进行跨站校验，必须校验

# 什么是 JSON ？

  1. JSON 指的是 JavaScript 对象表示法（JavaScript Object Notation）
  2. JSON 是轻量级的文本数据交换格式
  3. JSON 独立于语言
  4. JSON 具有自我描述性，更易理解

JSON 使用 JavaScript 语法来描述数据对象，但是 JSON 仍然独立于语言和平台。JSON 解析器和 JSON 库支持许多不同的编程语言。

# JSON和XML的比较

JSON 格式于2001年由 Douglas Crockford 提出，目的就是取代繁琐笨重的 XML 格式。

JSON 格式有两个显著的优点：书写简单，一目了然；符合 JavaScript 原生语法，可以由解释引擎直接处理，不用另外添加解析代码。所以，JSON迅速被接受，已经成为各大网站交换数据的标准格式，并被写入ECMAScript 5，成为标准的一部分。

XML和JSON都使用结构化方法来标记数据

# AJAX简介

AJAX（Asynchronous Javascript And XML）翻译成中文就是 “异步的Javascript和XML”

即使用Javascript语言与服务器进行异步交互，传输的数据为XML（当然，传输的数据不只是XML）。

AJAX 不是新的编程语言，而是一种使用现有标准的新方法。

AJAX 最大的优点是在不重新加载整个页面的情况下，可以与服务器交换数据并更新部分网页内容。（这一特点给用户的感受是在不知不觉中完成请求和响应过程）

AJAX 不需要任何浏览器插件，但需要用户允许JavaScript在浏览器上执行。

  1. 同步交互：客户端发出一个请求后，需要等待服务器响应结束后，才能发出第二个请求；
  2. 异步交互：客户端发出一个请求后，无需等待服务器响应结束，就可以发出第二个请求。

# AJAX 常见应用情景

搜索引擎根据用户输入的关键字，自动提示检索关键字。

还有一个很重要的应用场景就是注册时候的用户名的查重。

其实这里就使用了AJAX技术！当文件框发生了输入变化时，使用AJAX技术向服务器发送一个请求，然后服务器会把查询到的结果响应给浏览器，最后再把后端返回的结果展示出来。

  1. 整个过程中页面没有刷新，只是刷新页面中的局部位置而已！
  2. 当请求发出后，浏览器还可以进行其他操作，无需等待服务器的响应！

当输入登录用户名后，把光标移动到其他表单项上时，浏览器会使用AJAX技术向服务器发出请求，服务器会查询用户是否存在，最终服务器返回true表示名为用户已经存在了，浏览器在得到结果后显示“用户名已被使用”。

  1. 整个过程中页面没有刷新，只是局部刷新了
  2. 在请求发出后，浏览器不用等待服务器响应结果就可以进行其他操作

# AJAX的优点

  1. AJAX使用JavaScript技术向服务器发送异步请求；
  2. AJAX请求无须刷新整个页面；
  3. 因为服务器响应内容不再是整个页面，而是页面中的部分内容，所以AJAX性能高；

# AJAX请求如何设置csrf_token

# 方式1 - 在ajax中返回的值带上csrf_token

通过获取隐藏的input标签中的csrfmiddlewaretoken值，放置在data中发送。

依赖：要在当前文件中要有{% csrf_token %}，这配置，js才能获取到csrfmiddlewaretoken值

$.ajax({
  url: "/cookie_ajax/",
  type: "POST",
  data: {
      csrfmiddlewaretoken: $('[name="csrfmiddlewaretoken"]').val(),
      name: 'msg',
      age: '73',
  },
    success: function (res) {
        console.log(res);
  }
})

# 方式2 - 在ajax的请求头中带上csrf_token

通过获取返回的cookie中的字符串 放置在请求头中发送。

依赖：要在当前文件中要有{% csrf_token %}，这配置，js才能获取到csrfmiddlewaretoken值

$.ajax({
  url: "/cookie_ajax/",
  type: "POST",
  headers: {"X-CSRFToken": $.cookie('csrftoken')},  // 从Cookie取csrftoken，并设置到请求头中
  data: {
      csrfmiddlewaretoken: $('[name="csrfmiddlewaretoken"]').val(),
      name: 'msg',
      age: '73',
  },
  success: function (data) {
    console.log(data);
  }
})

# 方式3 - 自己写个js文件，需要时导入(官方推荐)

写一个js文件，包含以下的代码，当html中需要使用ajax跟后端交互的，就可以导入这个文件，就不用使用以上二种方式，这也是官方推荐的使用方式

依赖：要在当前文件中要有{% csrf_token %}，这配置，js才能获取到csrfmiddlewaretoken值

function getCookie(name) {
    var cookieValue = null;
    if (document.cookie && document.cookie !== '') {
        var cookies = document.cookie.split(';');
        for (var i = 0; i < cookies.length; i++) {
            var cookie = jQuery.trim(cookies[i]);
            // Does this cookie string begin with the name we want?
            if (cookie.substring(0, name.length + 1) === (name + '=')) {
                cookieValue = decodeURIComponent(cookie.substring(name.length + 1));
                break;
            }
        }
    }
    return cookieValue;
}
var csrftoken = getCookie('csrftoken');

function csrfSafeMethod(method) {
  // these HTTP methods do not require CSRF protection
  return (/^(GET|HEAD|OPTIONS|TRACE)$/.test(method));
}

$.ajaxSetup({
  beforeSend: function (xhr, settings) {
    if (!csrfSafeMethod(settings.type) && !this.crossDomain) {
      xhr.setRequestHeader("X-CSRFToken", csrftoken);
    }
  }
});

# 设置以上方式不用依赖cookie中的csrftoken

如果使用从cookie中取csrftoken的方式，需要确保cookie存在csrftoken值。

如果你的视图渲染的HTML文件中没有包含 {% csrf_token %}，Django可能不会设置CSRFtoken的cookie。

这个时候需要使用ensure_csrf_cookie()装饰器强制设置Cookie。

from django.views.decorators.csrf import ensure_csrf_cookie

@ensure_csrf_cookie  ## 这样子，返回的函数会让当前页面强制加上csrftoken值的cookie
def login(request):
    return redirect('/author/')

def upload(request):
    if request.is_ajax():  ## 判断请求是不是ajax请求，如果是以下代码
        file = request.FILES.get("file")  ## 获取到上传文件的信息
        with open(file.name, 'wb') as f:  ## 打开生成一个文件，file.name为上传文件数据中自带的原生文件名
            for i in file.chunks():  ## 使用for循环，慢慢的读取文件的数据
                f.write(i)  ## 写入
        return HttpResponse("OK")  ## 返回上传OK

    return render(request, "upload.html") ## 如果请求不是ajax，就返回页面

# ajax用户名是否已被注册示例

功能介绍

在注册表单中，当用户填写了用户名后，把光标移开后，会自动向服务器发送异步请求。服务器返回这个用户名是否已经被注册过。

案例分析

  1. 页面中给出注册表单；
  2. 在username input标签中绑定onblur事件处理函数。
  3. 当input标签失去焦点后获取 username表单字段的值，向服务端发送AJAX请求；
  4. django的视图函数中处理该请求，获取username值，判断该用户在数据库中是否被注册，如果被注册了就返回“该用户已被注册”，否则响应“该用户名可以注册”。

        return HttpResponse('用户名已存在')  ## 返回给前端的ajax
    else:
        return HttpResponse('用户名可以使用') ## 返回给前端的ajax

# 后端文件

from django.http import JsonResponse
from django.views import View
class Login(View):

    def get(self, request):  ## 接收来自get方式的请求
        return render(request, 'login.html')  ## 返回login页面

    def post(self, request):  ## 接收来自post方式的请求 == 接收ajax交互
        ret = {'status':0,'msg':''}  ## 定义一个字典，status：状态，msg：存储错误提醒
        user = request.POST.get('user')  ## 获取POST方式中的key为user的value数据
        pwd = request.POST.get('pwd')  ## 获取POST方式中的key为pwd的value数据
        obj = models.User.objects.filter(name=user, pwd=pwd).first() ## 进行数据库校验，first()取第一个，为对象
        if obj:  ## 判断是否有对象，如果有：证明校验成功，如果没有：证明校验失败
            ret['url'] = '/index/'  ## 向定义好的字典添加新的值，key：url，value：校验成功要返回的网址
            return JsonResponse(ret)  ## 通过json方式自动序列化字典数据传递回前端，到前端后自动反序列化

        else:  ## 证明校验失败
            ret['status']=1  ## 定义状态为1，1：失败，默认0：成功
            ret['msg']='用户名或密码错误' ## 定义错误提醒信息
            return JsonResponse(ret) ## 通过json方式自动序列化字典数据传递回前端，到前端后自动反序列化
