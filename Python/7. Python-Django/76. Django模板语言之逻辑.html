# 76. Django模板语言之逻辑

# for循环

# for循环常用的参数

Variable Description
forloop.counter 当前循环的索引值（从1开始）
forloop.counter0 当前循环的索引值（从0开始）
forloop.revcounter 当前循环的倒序索引值（从1开始）
forloop.revcounter0 当前循环的倒序索引值（从0开始）
forloop.first 当前循环是不是第一次循环（布尔值）
forloop.last 当前循环是不是最后一次循环（布尔值）
forloop.parentloop 本层循环的外层循环，包含以上的所有参数，已字典类型存储

# for循环支持的判断符

  1. and 、or、==、>、<、!=、<=、>=、in、not in、is、is no

# 简单for循环

<ul>
{% for user in user_list %}
    <li>{{ user.name }}</li>
{% endfor %}
</ul>

# for循环使用empty - 如果循环得到的值为空就执行

<ul>
{% for user in user_list %}
    <li>{{ user.name }}</li>
{% empty %}
    <li>空空如也</li>
{% endfor %}
</ul>

# 实例

{# up.html文件 #}
<table border="1">
    <tbody>
    {% for row in l1 %}
        <tr>
            {% for name in row %}
                {% if forloop.counter|divisibleby:'2' and forloop.parentloop.counter|divisibleby:'2' %}
                    <td style="color: green">{{ name }}</td>
                {% else %}
                    <td>{{ name }}</td>
                {% endif %}
            {% endfor %}
        </tr>
    {% endfor %}
    </tbody>
</table>

# if判断

# if判断支持的判断符

  1. and 、or、==、>、<、!=、<=、>=、in、not in、is、is not

# 单if判断

{% if user_list %}
  用户人数：{{ user_list|length }}

# if判断加else

{% if user_list|length > 5 %}
  七座豪华SUV
{% else %}
    黄包车
{% endif %}

# if判断 加else 加elif

{% if user_list %}
  用户人数：{{ user_list|length }}
{% elif black_list %}
  黑名单数：{{ black_list|length }}
{% else %}
  没有用户
{% endif %}

# with别名

定义一个中间变量，多用于给一个复杂的变量起别名。

注意等号左右不要加空格。

有二种起别名的方式

第一种

{% with total=business.employees.count %}
    {{ total }} employee{{ total|pluralize }}
{% endwith %}

第二种

{% with business.employees.count as total %}
    {{ total }} employee{{ total|pluralize }}
{% endwith %}

# 模板语言不支持连续判断

Django的模板语言不支持连续判断，即不支持以下写法：

{% if a > b > c %}
...
{% endif %}

但是可以这样写，以下的写法模板语言支持

{% if a > b and b > c %}
...
{% endif %}

# 模板语言中属性的优先级大于方法

Django的模板语言中属性的优先级大于方法

def xx(request):
    d = {"a": 1, "b": 2, "c": 3, "items": "100"}
    return render(request, "xx.html", {"data": d})

如上，我们在使用render方法渲染一个页面的时候，传的字典d有一个key是items并且还有默认的 d.items() 方法，此时在模板语言中:

{{ data.items }}

默认会取d的items key的值。

# 母版继承

母版就是我们在页面就，把公用的区域拿出来，放在一个单独的文件中，这个文件就可以被称为母版

注意：我们通常会在母板中定义页面专用的CSS块和JS块，方便子页面替换

# 继承母板 - extends

在子页面中在页面最上方使用下面的语法来继承母板。

{% extends 'layouts.html' %}

{#
	{% extends '母版文件' %}
#}

当程序执行到 {% extends 'layouts.html' %} 时，程序会自动把母版文件全部取出，就当成是当前文件的内容，进行展示

# 块 - block

块就是用于，在母版中，不是公共区域的地方，加上块逻辑，在子页面中，调用属于这子页面的块逻辑，进行编写，这样程序运行中，就会把子页面的块逻辑中的内容跟母版中的块逻辑进行替换

通过在母板中使用 {% block page-main %} 来定义"块"。

在子页面中通过定义母板中的block名来对应替换母板中相应的内容。

{% block page-main %}
  <p>12321</p>
  <p>321312</p>
  <p>3242342</p>
{% endblock %}

# 组件 - include

可以将常用的页面内容如导航条，页尾信息等组件保存在单独的文件中，然后在需要使用的地方按如下语法导入即可。

{% include 'navbar.html' %}

# 静态文件相关

# {% static '路径' %}

{% static '路径' %}，的作用就是自动获取STATIC_URL的目录名，在跟旁边的路径进行拼接，变成一个完整路径

基本用于，静态文件的路径问题，这样子设置就可以把路径变成动态路径，就算去使用固定目录名，也不用一个一个文件的修改路径了

{% load static %}
<img src="{% static "images/hi.jpg" %}" alt="Hi!" />

引用JS文件时使用：

{% load static %}
<script src="{% static "mytest.js" %}"></script>

某个文件多处被用到可以存为一个变量

{% load static %}
{% static "images/hi.jpg" as myphoto %}
<img src="{{ myphoto }}"></img>

# "{% get_static_prefix %}路径"

跟上面的static类似，但是get_static_prefix只会帮你自动获取STATIC_URL的目录名，不会帮你拼接路径， 所以一般使用这个都是在后面加上路径，这样结果默认也形成了完整的路径

基本用于，静态文件的路径问题，这样子设置就可以把路径变成动态路径，就算去使用固定目录名，也不用一个一个文件的修改路径了

一般都是用static

{% load static %}
<img src="{% get_static_prefix %}images/hi.jpg" alt="Hi!" />

{% load static %}
{% get_static_prefix as STATIC_PREFIX %}

<img src="{{ STATIC_PREFIX }}images/hi.jpg" alt="Hi!" />
<img src="{{ STATIC_PREFIX }}images/hi2.jpg" alt="Hello!" />

# 自定义simple_tag

创建的步骤跟自定义filter一模一样，存放文件的目录都一样

和自定义filter类似，只不过接收更灵活的参数。

定义注册simple tag

@register.simple_tag(name="plus")
def plus(a, b, c):
    return "{} + {} + {}".format(a, b, c)

使用自定义simple tag

{% load demo %}

{# simple tag #}
{% plus "1" "2" "abc" %}

# inclusion_tag

创建的步骤跟自定义filter一模一样，存放文件的目录都一样

inclusion_tag的作用：当程序调用时，就会先去找所属函数，所属函数按逻辑对指定的html文件进行修改，在返回这个html给程序

创建一个inclusion_tag函数

from django import template

register = template.Library()


@register.inclusion_tag('result.html')  ## （）括号中，写的一个静态文件名，
def show_results(n):
    n = 1 if n < 1 else int(n)
    data = ["第{}项".format(i) for i in range(1, n+1)]
    return {"data": data}

html文件

<ul>
  {% for choice in data %}
    <li>{{ choice }}</li>
  {% endfor %}
</ul>

调用

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta http-equiv="x-ua-compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>inclusion_tag test</title>
</head>
<body>

{% load inclusion_tag_test %}  {# 导入 #}

{% show_results 10 %}  {# 调用inclusion_tag中的show_results，并把10数据传递过去 #}
</body>
</html>

# 实例

一个分页小逻辑，让页面自己控制分页的页面数，返回当前的页面数值等

nclusion_tag函数

from django import template

register = template.Library()


@register.inclusion_tag('pagination.html')
def pagination(total, current_num):
    return {'total': range(1, total + 1), 'current_num': current_num}

html文件

<div>
    <nav aria-label="Page navigation" class="pull-right">
        <ul class="pagination">
            <li>
                <a href="#" aria-label="Previous">
                    <span aria-hidden="true">&laquo;</span>
                </a>
            </li>
            {% for num in total %}
                {% if num == current_num %}
                    <li class="active"><a href="#">{{ num }}</a></li>
                {% else %}
                    <li><a href="#">{{ num }}</a></li>
                {% endif %}
            {% endfor %}
            <li>
                <a href="#" aria-label="Next">
                    <span aria-hidden="true">&raquo;</span>
                </a>
            </li>
        </ul>
    </nav>
</div>

调用主要代码

<!--分页开始-->
{% load mytags %}
{% pagination 20 10 %}
<!--分页结束-->

{# 
	传递二个参数
		第一个为分页页面数
		第二个为当前页数
#}
